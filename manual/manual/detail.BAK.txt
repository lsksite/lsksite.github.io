<H1>The Detail screen contains the data for the main body of the invoice.</H1>
<HR>
<H2>Completing the Detail screen.</H2>
<P>(A Header must be created before starting the Details screen. Please
be sure to review <A HREF="header.htm">HEADER</A> before continuing with this page.)</P>
<P>The example we are working with is an invoice for a shipment to one of our subsidiaries against
an Oracle sales order. After completing the items discussed in the Header section it will then
be necessary to download the order details from Oracle.</P>
<P>With the Header window saved and still visible, click on the download button
<IMG SRC="hp.gif"align=top>. This will bring up the Download Parameters box:
<p><CENTER><IMG SRC="dwnld02.gif"align=middle hspace=20><B><TT><A NAME="DP">DOWNLOAD PARAMETERS</A> dialog box</TT></B></CENTER></P>
<P>Click on <strong>Orders</strong>, then <strong>OK</strong>. This will
start the download process from Oracle to the IIG database. (Remember, the key field in the Header to enable the correct data to be downloaded is the 
<A HREF="header.htm#Ref">Reference</A> field). The Detail window will appear (see top of page) 
with data downloaded to the following fields: <strong>Line#, Model#, Description, Qty, Unit Cost,
Unit Price</strong> and <strong>Tariff Item#.</strong> (Note that data downloaded into the <strong>Tariff Item#</strong> field
and the <strong>Unit Cost</strong> field may not always be complete. This data does not come directly from the Oracle Sales Order but from
fields in Oracle that are linked to the Model Code so these fields need to be reviewed
for completeness and accuracy). All data fields in the Detail window mentioned above can be altered manually and, in the
case of <strong>Tariff Item#</strong>, with the drop-down menu. Note that fields with blue backgrounds
(<strong>Ext Cost x Tran Rate</strong> and <strong>Ext Price</strong>) are calculated fields and cannot
be updated directly.</P>
<P>It's very important to understand the difference between <strong>Unit Cost</strong> and <strong>
Unit Price</strong>. As was briefly mentioned in the Header tutorial, the selection of the country
from the <A HREF="header.htm#Cntry">drop-down country list</A> has a direct impact on the pricing on the invoice.
When we ship product to our subsidiaries we currently use standard cost times whatever transfer
rate has been established. When we ship to an end-user (distributor) we use the sales price
that OA has entered for that particular sales order. Currently, the way the Generator determines
what pricing scheme to use is by the <strong>Country Code</strong> that was entered in the Header.
When a particular country is intitially setup in the Generator (and a <strong>Country Code</strong> established)
it must be determined at that point whether to use <strong>Unit Cost</strong> or <strong>Unit Price</strong> for shipments to that
country. In situations where shipments are possible, within the same country, to our subsidiary or
to an end-user it is necessary to setup <strong>two separate country codes</strong>: one for <strong>Unit Cost</strong>
(transer price) and one for <strong>Unit Price</strong>.</P>
<P>It's extremely important to understand how the Generator distinguishes between these two
pricing schemes. The final invoice will be wildly wrong if the incorrect pricing was used. Take note of the data
in the bottom part of the Detail window:</P>
<P><center><IMG SRC="detail03.gif"></center></P>
<P>If there is data showing in the <strong>Total Value in U.S. Dollars</strong> field then the invoice
amounts will be in <strong>Unit Cost</strong> (x Trans Rate). If data is showing in the <strong>Tot Price</strong>
field then the invoice amounts will be in <strong>Unit Price</strong>. If data is in the incorrect field
then the <strong>Country Code</strong> will need to be changed in the Header screen to reflect the proper
pricing scheme. Also note that if the invoice has been setup to use <strong>Unit Price</strong> there will
still be an amount indicated for the Transfer Rate (Tran Rate). This number will be <strong>ignored</strong> in the final
output on the Invoice, so don't be fooled by thinking that if there is a Transfer Rate indicated in the 
Detail window that the invoice will be using <strong>Unit Cost</strong> because it may not be. The example above
does indicate that The Generator will be using <strong>Unit Cost</strong> (x Tran Rate) for the final output.</P>
<P>It's a good idea to save the Detail screen immediately after downloading and after any changes are made.
If there are any backorders, the <strong>Qty</strong> field can be changed or the whole line can be deleted
whichever is appropriate. To delete a line make sure any field in that line is highlighted then
click on the Trash Can button <IMG SRC="trash.gif"align=top>.</P>
<P>If a line needs to be added to an invoice click on the Add New Line button <IMG SRC="insert.gif"align=top>.
This will create a new line in the Detail window where data can be entered manually. Alternatively, if additional
items need to be added to an invoice they can be downloaded separately by using the <strong>Item Master</strong>
option in the <A HREF="#DP">Download Parameters</A> dialog box. Click on the Download button <IMG SRC="hp.gif"align=top>
then click on the Item Master option in the Download Parameters box. Another box will appear where the additional
item(s) can be entered. Click OK and the new item(s) along with their Unit Cost and Description will be added to the Detail window.
The Qty for these additional items will then need to be entered manually. Be sure to SAVE after any changes are made.</P>
<P>Note that when the <strong>Item Master</strong> option is used for downloading it will always <strong>add</strong> to what has
already been entered in the Detail screen. When the <strong>Orders</strong> option is used it will always <strong>overwrite</strong>
what was previously downloaded. Also it's important to note that when the <strong>Orders</strong> option is used it will also
enter data to the Header screen as was explained earlier in the <A HREF="header.htm#Oracle">Header tutorial</A>. If this data
was previously entered manually to the Header screen Oracle will overwrite it. To maintain the data that was entered
manually make sure not to save the Header window after the Oracle order is downloaded. This is done by not clicking on save
when the Header screen is ACTIVE. Data can be saved many times while the Detail screen is active without affecting
the Header, but if the Header screen is saved after an Oracle order download then the manually entered data will be permanently overwritten.</P>
<P>After all the necessary items have been downloaded and the Detail window reviewed for accuracy (and Saved!) it is time to
proceed to the final element of the invoice: The <A HREF="footer.htm">Footer</A> window.</P>
